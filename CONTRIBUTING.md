# Contributing Guidelines

Thank you for your interest in contributing to this project!

## Code of Conduct

This project follows the [CNCF Code of Conduct](https://github.com/cncf/foundation/blob/main/code-of-conduct.md). By participating, you are expected to uphold this code.

## How to Contribute

### Reporting Issues

- Check existing issues before creating a new one
- Use the issue templates when available
- Provide clear, detailed descriptions
- Include reproduction steps for bugs

### Submitting Changes

1. Fork the repository
2. Create a feature branch from `staging`
3. Make your changes
4. Write or update tests as needed
5. Ensure all tests pass
6. Submit a pull request to `staging`

### Branch Strategy

- `main` - Production-ready code
- `staging` - Integration and testing
- Feature branches - Individual contributions

### Commit Messages

Write clear, descriptive commit messages:

```
Short summary (50 chars or less)

More detailed explanation if needed. Wrap at 72 characters.
Explain what and why, not how.

- Bullet points are acceptable
- Use present tense ("Add feature" not "Added feature")
```

### Pull Request Guidelines

- Reference related issues
- Provide a clear description of changes
- Include screenshots for UI changes
- Ensure CI checks pass
- Request review from maintainers

## Development Setup

Instructions for setting up a local development environment will be documented as the project develops.

## Questions?

Open an issue with the "question" label for any clarifications.
